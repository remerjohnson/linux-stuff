Full Collection Description:
This repository contains the molecular dynamics trajectories scripts and models used in the paper “Allostery through the computational microscope: cAMP activation of a canonical signaling domain”. The purpose of this repository is to share the simulation data used to build the models and to show how the models presented in the paper were constructed and analyzed. This data repository contains three data sets 
 
 S00-MDScripts.tar.gz
 
 The first data set contains the files required to perform molecular dynamics simulations from scratch using the Amber molecular dynamics software package, not included. This unit contains the parameter and initial coordinate files need to simulate the four starting systems, fully solvated. The four systems representing both the active and inactive conformations of the cyclic-nucleotide binding domain with and without cAMP bound.  Additionally, the data set contains the input files used with Amber to perform system minimization, equilibration, and production run. Amber can be obtained at ambermd.org.
 
 S01-MD_Trajectories.tar
 
 The second data set contains the molecular dynamics trajectories used to  construct the Markov state models presented in the paper. The trajectories are presented in the .dcd format, and contain only the atomic coordinates for the protein studied (i.e. no solvent or ligand). The corresponding .pdb file required for visualization using a program like VMD (www.ks.uiuc.edu/Research/vmd/) are included. Each trajectory is labeled as follows:
 
 Starting_system-system_state-simulation_run-rep_number.alpro.dcd
 
 With starting_system identifies which family of simulations trajectory belongs, either RAIaB or RAIaH, and system_state identifies whether cAMP was bound to the system during the simulations. System_state-simulation identifies the type and architecture used for each molecular dynamics simulation with AS identifying Anton simulations, IS identifying initial sampling simulations using GPU accelerated Amber, and PC identifying adaptive sampling runs using GPU accelerated Amber. Rep_number indicates an independent sampling run with unique initial velocities. Due to the large size of the Anton trajectories these have been broken into multiple sets.
 
 
 
 
 S02-Model_Building_Work_and_Scritps.tar.gz
 
 The third data set is effectively the “lab notebook” of the Markov state models, analysis, and scripts utilized in the associated paper. It contains snapshot of the scripts used and analyses performed in the paper.  It also contains many of intermediate models and alternate analyses not discussed in the paper. The objective of this data set is not to provide a step-by-step procedure to reproduce the work  presented in the paper but to be the equivalent of  "showing your the work”.  Specific questions regarding scripts or intermediate models can be directed to the authors of the associated paper. 
 
 Many of the Python scripts included in the data set are dependent on libraries not included in this data set, but which are publicly available.  Additionally, this work was done with MSMBuilder2 available through SimTk.org, which is not included in this package.
